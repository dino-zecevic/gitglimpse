# gitglimpse PR Context Action

Automatically generate structured PR descriptions from your git history.

## Usage

Add to your workflow:

```yaml
name: PR Context
on:
  pull_request:
    types: [opened, synchronize]

jobs:
  context:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0
      - uses: dino-zecevic/gitglimpse@main
        with:
          github-token: ${{ secrets.GITHUB_TOKEN }}
```

## Inputs

| Input | Description | Default |
|-------|------------|---------|
| github-token | GitHub token for posting comments | ${{ github.token }} |
| base-branch | Branch to compare against | main |
| context-mode | Detail level: commits, diffs, both | both |
| filter-noise | Filter merge/lint/lock commits | true |

## What it does

On every PR, gitglimpse:
1. Analyzes the commits on your branch
2. Groups them into logical tasks
3. Extracts ticket IDs from branch names
4. Filters noise (merges, lock files, formatting)
5. Posts a structured context summary as a PR comment

The comment updates on each push — no duplicate comments.
